# Requisitos Funcionais (RF)
## Usuário

[x] RF01: O sistema deve permitir cadastro de usuário
[x] RF02: O sistema deve permitir login
[] RF03: O sistema deve permitir autenticação via token (JWT)

## Tarefas

[] RF04: O usuário deve poder criar uma tarefa
[] RF05: O usuário deve listar suas tarefas
[] RF06: O usuário deve visualizar detalhes de uma tarefa
[] RF07: O usuário deve editar uma tarefa
[] RF08: O usuário deve concluir uma tarefa
[] RF09: O usuário deve excluir uma tarefa
[] RF10: O usuário deve filtrar tarefas por status
[] RF11: O usuário deve ordenar tarefas por prioridade ou data

## ⚙️ Requisitos Não Funcionais (RNF)

[] RNF01: A API deve seguir o padrão REST
[] RNF02: A aplicação deve usar autenticação JWT
[] RNF03: O backend deve ser desenvolvido em TypeScript
[] RNF04: O frontend deve ser desenvolvido em React
[] RNF05: A API deve validar os dados de entrada
[] RNF06: O sistema deve proteger rotas autenticadas
[] RNF07: A aplicação deve ter código organizado por camadas
[] RNF08: A senha deve ser armazenada de forma segura (hash)
